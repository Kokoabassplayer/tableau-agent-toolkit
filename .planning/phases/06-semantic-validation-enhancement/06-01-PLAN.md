<objective>
Enhance the SemanticValidator to map semantic errors back to spec line numbers and provide remediation guidance.

Purpose: When a semantic validation error is found in a generated TWB, the user should see exactly which line in their dashboard_spec.yaml caused the problem and what to do about it. This closes SPEC-04 (spec line mapping) and VAL-03 (remediation steps).

Output: Updated SemanticIssue dataclass with new fields, rewritten _build_spec_index with yaml.compose() line tracking, REMEDIATION_MAP constant, and comprehensive unit tests.
</objective>

    **Step 3: Rewrite _build_spec_index in semantic.py**

    Replace the existing `_build_spec_index` method entirely. The new implementation:

    a) Change the return type from `dict[str, str]` to `dict[str, tuple[str, int]]` -- each entry maps an element key to (spec_path_fragment, 1-based_line_number).

    b) Use `yaml.compose()` (import as `yaml_module` to avoid collision with any local `yaml` variable) to build a Node tree with line numbers.

    c) Walk the Node tree for sections: "worksheets", "datasources", "calculations" (each is a SequenceNode of MappingNodes with a "name" key).

    d) Walk "dashboards" section: for each dashboard MappingNode, find the "sheets" key (SequenceNode of ScalarNodes) and index each sheet name string. The key format is `zone:{sheet_name}` and the value is `(dashboards[{i}].sheets[{j}], line)`.

    e) Handle edge cases: stream is None (empty file) -> return {}, stream is not MappingNode -> return {}, OSError on file read -> return {}.

    f) Line numbers: always `node.start_mark.line + 1` (PyYAML is 0-based, display is 1-based).

    g) Update the `spec_index` type annotation in `validate()` from `dict[str, str]` to `dict[str, tuple[str, int]]`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML file -> yaml.compose() | User-provided YAML parsed into Node tree; malformed YAML could raise YAMLError |
| Filesystem -> spec_path.read_text() | Reading spec file from disk; file may not exist or be unreadable |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | D (Denial of Service) | yaml.compose() on user YAML | accept | Spec files are small (<100KB). yaml.compose() loads full tree but this is acceptable for our use case. No entity expansion risk with YAML. |
| T-06-02 | T (Tampering) | _build_spec_index file read | mitigate | Wrap spec_path.read_text() in try/except OSError; return empty dict on failure. No crash on malformed/missing files. |
</threat_model>
