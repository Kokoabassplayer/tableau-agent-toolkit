Create src/tableau_agent_toolkit/twb/generator.py with:

1. `GenerationResult` dataclass:
   - `output_path: Path`
   - `warnings: list[str]`

2. `WorkbookGenerator` class:
   - `__init__(self, template_registry: TemplateRegistry)`:
     - Store template_registry
   - `generate(self, spec: DashboardSpec, output_path: Path, template_override: Path | None = None) -> GenerationResult`:
     - If template_override is provided, use that path; otherwise resolve via template_registry.resolve(spec.workbook.template.id, spec.workbook.target_tableau_version)
     - Parse template with `etree.parse(str(template_path))` using `etree.XMLParser(remove_blank_text=True)` for clean formatting
     - Call `apply_manifest_by_version(tree, spec.workbook.target_tableau_version)`
     - Call `self._patch_workbook_name(tree, spec.workbook.name)`
     - Call `self._patch_datasources(tree, spec)`
     - Call `self._patch_calculations(tree, spec)`
     - Call `self._patch_worksheets(tree, spec)`
     - Call `self._patch_dashboards(tree, spec)`
     - Create parent directories: `output_path.parent.mkdir(parents=True, exist_ok=True)`
     - Write: `tree.write(str(output_path), encoding="utf-8", xml_declaration=True)`
     - Return GenerationResult(output_path=output_path, warnings=[])

   - `_patch_workbook_name(tree, name: str) -> None`:
     - Set root element's "name" attribute to name value
     - Also find or create <preferences> element and set <preferences><preference name="ui.show-workbook-tabs" value="true" />

   - `_patch_datasources(tree, spec: DashboardSpec) -> None`:
     - Find or create `<datasources>` element under root
     - For each datasource in spec.datasources: create a `<datasource>` element with name, caption, and inline attributes
     - For custom-sql mode: add `<connection>` element with custom SQL reference

   - `_patch_calculations(tree, spec: DashboardSpec) -> None`:
     - For each calculation in spec.calculations: find the appropriate datasource and add `<column>` element with datatype="string", name=calculation.name, role="measure" or "dimension", and `<calculation>` child with formula attribute

   - `_patch_worksheets(tree, spec: DashboardSpec) -> None`:
     - Find or create `<worksheets>` element under root
     - For each worksheet in spec.worksheets: create a `<worksheet>` element with name attribute

   - `_patch_dashboards(tree, spec: DashboardSpec) -> None`:
     - Find or create `<dashboards>` element under root
     - For each dashboard in spec.dashboards: create a `<dashboard>` element with name attribute and a basic `<zones>` child

Use `from lxml import etree` for all XML operations. Use `resolve_entities=False` and `no_network=True` in XMLParser for security when parsing templates.
